? Help & Known Limitations

File Classification

Cabinet vs Desktop detection is filename-only. The app reads keywords like _CAB, _Cabinet, _Desktop, _4K from the filename. Files without these labels are treated as universal. Modern VPW releases label themselves correctly; older tables are generally universal.

ZIPs with no recognised content are classified as ROM. If a ZIP contains nothing the app recognises (no VPX, B2S, POV, media, or PuP Pack markers) it is moved to VPinMAME\roms\. Move it manually if that's wrong.

VPU Remix Patches (.dif) are manual-only. ZIPs containing .dif files need the external VPUPatcher64 tool from vpuniverse.com. The app shows instructions but cannot apply these patches itself.

PinUp Popper Integration

Close Popper before installing. Writing to PUPDatabase.db while Popper is running can corrupt the database. The app will prompt you to confirm Popper is closed.

Registration matching may miss non-standard filenames. Detection compares a normalised version of the VPX filename against what Popper stored. If Popper's entry was manually renamed to something completely different, the "not in Popper" badge may be wrong. Re-scan Popper DB after any manual edits.

SQL fallback requires DB Browser for SQLite. If the Popper DB path isn't set, the app generates a popper_import.sql file. Apply it via DB Browser for SQLite → Execute SQL tab → F5 → Write Changes. Always back up the DB first.

PuP Packs

The 🎮 PuP Pack button is for extracted folders only. Use it to point at an already-extracted PuP Pack folder (e.g. Aladdin\). For PuP Pack ZIP files, use 📥 Install Files instead — ZIPs are auto-detected and extracted automatically.

Patch ZIPs merge into existing folders. ZIPs whose name contains "patch" or "update" are merged into the existing PUPVideos\{tablename}\ folder, overwriting files with the same name.

Path Setup

Auto-detect needs the right root folder. Pick a folder that contains both VisualPinball and PinUPSystem as direct or nested subfolders — typically C:\vPinball. If you pick a subfolder like Tables or roms, most paths won't be found.

Scanning & Detection

Re-scan after installing. Installed badges, version detection, duplicate detection, and Popper status are all based on the last scan. After installing new files click Re-scan to refresh everything.

ROM name in table detail uses VPS data, not per-version data. VPS stores one ROM entry per table — not one per VPX version. If a newer VPX version needs a different ROM revision than VPS lists, the detail panel will show the stale value. The install plan compensates automatically when you drop a ROM alongside the VPX.

Binary version extraction covers 95%+ of tables. The app reads cVersion from the first 4 MB and last 2 MB of each VPX file. Tables where the script falls outside these regions fall back to filename-based version detection.

General

"Recently Updated" uses VPS timestamps, not your install date. A table you installed 2 years ago can appear as recently updated if someone uploaded a new version this week.

Duplicate detection uses compact name matching. Two VPX files are considered duplicates if their names reduce to the same string after stripping version numbers, years, and punctuation. Check both filenames before deleting.

Backup rotation keeps only 3 copies. PinballMagic\Backups\ auto-prunes to the 3 most recent PUPDatabase.db backups. Copy older backups elsewhere if you need to keep more.

"no bg media" badge is informational only. It means no backglass video in Popper's POPMedia\BackGlass\ — this is separate from the .directb2s file. Popper media videos come from VPinUniverse and cannot be downloaded from within this app. The table will play correctly regardless.

Data & Privacy

Anonymous download counts power 🔥 Hot Right Now. When you download a table, PinballMagic logs the VPS table ID, an anonymous device ID (randomly generated, never tied to your account or IP), and the file type to a shared database. This data is aggregated to show which tables are most popular across all users.

No personal information is collected. The anonymous device ID is a random string stored only on your device. No name, email, IP address, or account is ever sent.

You can opt out any time. Go to Settings → Privacy and turn off "Share anonymous download data." Existing data is not deleted but no new records will be sent.
